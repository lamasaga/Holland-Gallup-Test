# 在线测评工具实施计划
# Online Assessment Tool Implementation Plan

> **版本 / Version**: 1.0  
> **定位 / Positioning**: 基于既有研究成果（RIASEC × CliftonStrengths 双维度评价体系），将在线测评工具从概念落地为可执行、可迭代的开发方案。  
> **范围 / Scope**: 涵盖学生端与老师端双端口、分次测评、报告生成、职业-专业映射等核心功能模块。

---

## 1. 项目目标 / Project Objectives

### 1.1 核心目标

1. 搭建一个支持 **学生端** 与 **老师端** 双角色登录的在线测评平台。
2. 学生可 **分两次完成** Holland RIASEC 兴趣测评与 Gallup CliftonStrengths 优势测评。
3. 老师可查看学生完成情况，并为单个学生生成 **学生版** 与 **专业版** 双版本解读报告。
4. 报告核心任务是建立 **测评结果 → 未来职业方向 → 可申请专业** 的映射关系，并支持 **多对多映射**。

### 1.2 用户价值

| 用户 | 价值 |
|-----|------|
| 学生 | 了解自己的兴趣舞台与优势风格，获得可探索的职业与专业方向 |
| 老师/咨询师 | 高效查看学生测评结果，生成结构化报告，支撑一对一职业咨询 |
| 学校/机构 | 沉淀学生测评数据，形成可追踪的生涯发展档案 |

## 5. 测评流程设计 / Assessment Flow

### 5.1 学生端流程

```
登录 → 选择角色“学生” → 进入仪表盘
                    ↓
        ┌───────────┼───────────┐
        ↓           ↓           ↓
   开始 Holland   开始 Gallup   查看报告
        ↓           ↓           ↑
   答题/保存进度  选择 Top 5     │
        ↓           ↓           │
   提交得三码    提交得领域      │
        └───────────┴───────────┘
                    ↓
            两个都完成后解锁报告
```

### 5.2 Holland 测评设计

- 题目数量：60 题（每类型 10 题）。
- 计分方式：5 点量表（1=很不喜欢，5=非常喜欢），同类型得分相加。
- 输出：六维度得分 + 三字母代码。
- 进度保存：每答完一页自动保存，支持断点续测。

### 5.3 Gallup 测评设计

#### 5.3.1 当前实现：180 题迫选版

基于用户提供的 Gallup 测评 OCR 文本，已完成 180 道迫选题的整理与校对，每道题均包含：

- **A/B 两个陈述**：描述两种不同行为/思维倾向。
- **场景化提示**：把抽象陈述还原到学习、生活、工作场景。
- **五个选项**：非常认同A、比较认同A、两者居中、比较认同B、非常认同B。

答题者根据第一直觉，在每道题的五个选项中选择最符合自己的一项。

#### 5.3.2 计分规则（简化版）

1. 每道题的 A/B 两个陈述分别对应若干 CliftonStrengths 主题（通过关键词启发式匹配预先生成主题-题目映射）。
2. 选项分值赋值：
   - 非常认同A = +2 分（A 侧主题）
   - 比较认同A = +1 分（A 侧主题）
   - 两者居中 = 0 分
   - 比较认同B = +1 分（B 侧主题）
   - 非常认同B = +2 分（B 侧主题）
3. 累加每道题对应主题的分值，得到 34 个主题的 raw score。
4. 按 raw score 从高到低排序，取 Top 5/Top 10。
5. 统计 Top 5 在四大领域中的分布，识别主导领域。

> **说明**：该计分方式为非官方简化算法，仅用于探索性分析。正式商用场景若使用 Gallup 完整版 177 对陈述，需获得 Gallup 官方授权并使用其官方计分算法。

#### 5.3.3 题目管理

- 题库文件：`data/processed/gallup_180_questions_list.md`
- 主题-题目映射：`data/processed/gallup_questions_students_association.md`
- 导入方式：系统初始化时从 Markdown/JSON 文件导入题库，存入数据库 `questions` 表。

| C5 | 使用办公软件和信息系统处理事务 | 你擅长用 Word、Excel、Notion 等工具处理信息、排日程、做统计 |
| C6 | 核对数字、检查细节、发现错误 | 你看书或看报表时，容易发现排版错误、计算错误或逻辑漏洞 |
| C7 | 建立分类系统或数据库 | 你喜欢给藏书、电影、游戏或资料建立分类标签和索引 |
| C8 | 编写规范、手册或操作指南 | 完成一项任务后，你愿意写一份清晰的使用说明或操作流程 |
| C9 | 管理日程、会议或行政事务 | 你擅长安排时间表、发会议通知、提醒 deadlines，让一切有条不紊 |
| C10 | 按照清单逐项完成重复性工作 | 完成一项项任务并打勾，会让你有满足感和掌控感 |

#### 6.1.3 计分规则

1. 每题得分 1-5 分。
2. 同类型 10 题得分相加，得到该类型总分（范围 10-50 分）。
3. 按总分从高到低排序，取前三位构成三字母代码。
4. 若出现同分，可同时显示并列类型，或按作答顺序优先。

#### 6.1.4 结果解释

| 代码位置 | 含义 |
|---------|------|
| 第一字母 | 主导兴趣类型：最吸引你的职业舞台 |
| 第二字母 | 表达方式：你如何实现主导兴趣 |
| 第三字母 | 辅助风格：进一步细化你的职业偏好 |

**示例**：
- **SAE**：社会型（主导）+ 艺术型 + 企业型 → 适合用创意和影响力的方式帮助他人，如艺术治疗师、教育培训机构创始人、非营利组织领导者。
- **IRC**：研究型（主导）+ 现实型 + 常规型 → 适合有结构、有实际应用的研究工作，如数据科学家、质量工程师、医学技术员。

#### 使用说明

1. **每道题均为迫选形式**：每题呈现为"A 陈述 vs. B 陈述"，答题者需要在五个认同度选项中选择一项。
2. **场景化解读仅作参考**：帮助学生把抽象陈述与真实行为联系起来，但不改变题目原意。
3. **题目完整性说明**：180 道题已全部补全、校对并添加场景化提示与五个选项，可直接用于系统开发。正式商用若使用 Gallup 完整版 177 对陈述，需获得 Gallup 官方授权。


#### 6.2.5 两种 Gallup 测评入口

**入口 A：180 题迫选版（推荐）**

1. 学生依次作答 180 道迫选题，每题在 A/B 两个陈述中选择认同度。
2. 系统自动计分，生成 34 主题排序。
3. 取 Top 5/Top 10，并统计四大领域分布。
4. 该入口更接近官方迫选体验，可导出完整主题剖面。

**入口 B：主题自评选择版（备用/快速体验）**

1. 学生阅读 34 个主题的定义。
2. 按“最像我”的顺序依次点击选择 5 个主题。
3. 第一个选择的主题为能量最突出主题，依次递减。
4. 系统根据主题所属领域统计分布，识别主导领域。

#### 6.2.6 主题与 RIASEC 的参考映射

> 本映射为理论推导，非 Gallup 或 Holland 官方发布，仅用于职业对话参考。

| CliftonStrengths 领域 | 主要对应 RIASEC 类型 | 次要对应 RIASEC 类型 |
|---------------------|---------------------|---------------------|
| Strategic Thinking | I 研究型、A 艺术型 | — |
| Executing | C 常规型 | R 现实型 |
| Relationship Building | S 社会型 | — |
| Influencing | E 企业型 | S 社会型 |

## 7. 报告生成逻辑 / Report Generation Logic

### 7.1 输入

- 学生 Holland 三字母代码：`holland_code`（如 `SAE`）
- 学生 CliftonStrengths Top 5：`gallup_top5`
- 主题 → 领域映射表

### 7.2 处理步骤

1. **提取主导类型**：取 Holland 代码第一字母。
2. **提取主导领域**：统计 Top 5 主题在四大领域中的分布，取数量最多的领域。
3. **定位职业矩阵**：在 `RIASEC × CliftonStrengths 领域` 矩阵中查找交叉象限的职业列表。
4. **专业映射**：根据每个推荐职业，查询 `career_major_mapping` 表，获取相关专业列表。
5. **张力分析**：根据主导类型与主导领域的组合，调用预设张力说明。
6. **生成双版本报告**：分别渲染学生版与专业版 HTML/PDF。

### 7.3 多对多映射说明

- 一个职业可对应多个专业（如“数据科学家”可对应数据科学、统计学、计算机科学、数学）。
- 一个专业也可通向多个职业（如“心理学”可通向心理咨询师、用户研究员、人力资源等）。
- 报告中需明确标注“推荐专业仅为参考路径，不是唯一答案”。

### 7.4 报告素材库调用

报告生成模块调用 `data/processed/cliftonstrengths_report_materials.json` 中的主题描述，为每个 Top 5/Top 10 主题渲染以下内容：

| 字段 | 在报告中的作用 |
|-----|--------------|
| `standard_definition` | 学生版报告：用一句话让学生理解该主题 |
| `feature` | 学生版/专业版：才干特征概述 |
| `description` | 专业版报告：详细才干描述 |
| `application` | 学生版报告：如何应用该优势 |
| `blind_spots` | 专业版报告：提醒注意过度使用或潜在风险 |

#### 报告模板结构（学生版）

1. **封面**：姓名、测评日期、Holland 三码、Gallup 主导领域。
2. **我的兴趣舞台**：Holland 三码解读 + 三个类型的具体描述。
3. **我的优势风格**：Top 5 主题 + 每个主题一句话定义 + 场景化例子。
4. **组合画像**：兴趣 × 优势交叉描述（如 SAE × 关系建立）。
5. **推荐方向**：3-5 个职业方向 + 每个方向对应可申请专业。
6. **下一步行动**：3 条具体可执行的探索建议。

#### 报告模板结构（专业版）

1. **测评结果摘要**：Holland 剖面图、Gallup 领域分布图、Top 5/Top 10 列表。
2. **RIASEC 剖面分析**：六维度得分、三码解释、并列/边缘类型说明。
3. **CliftonStrengths 主题叙事**：Top 5 详细描述、主题组合故事线、主导领域分析。
4. **兴趣-优势整合分析**：RIASEC 主导类型与 CliftonStrengths 主导领域的契合度与潜在张力。
5. **职业-专业映射表**：推荐职业、对应专业、证据等级、适配理由。
6. **发展建议与伦理提醒**：优势应用建议、盲点管理、使用边界声明。

## 8. 技术方案 / Technical Stack

### 8.1 推荐方案

| 层级 | 技术选型 | 说明 |

## 12. 风险与限制 / Risks & Limitations

### 12.1 主要风险

| 风险 | 影响 | 应对策略 |
|-----|------|---------|
| 测评工具信度争议 | 结果被误用为高利害决策 | 在报告中明确证据等级与使用边界 |
| 职业-专业映射不准确 | 误导学生专业选择 | 标注“参考路径”，结合地区院校数据 |
| 数据隐私泄露 | 学生敏感信息外泄 | 加密存储、权限隔离、最小化数据收集 |
| Gallup 版权 | 完整版 177 题可能涉及商业授权 | 当前使用自研 180 题迫选版；正式商用若使用 Gallup 官方题库，需获得授权 |

### 12.2 使用限制

- 不用于选拔、分班、评奖等 high-stakes 决策。
- 不替代专业心理咨询或临床评估。
- 职业建议仅为可能性探索，需结合能力、价值观、市场需求综合判断。
