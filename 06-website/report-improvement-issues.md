## 二、问题清单

### 1. [高] 艺术型（A）结果可能引发就业焦虑 ✅ 已实施

- **类别 / Category**：用户心理
- **问题描述 / Description**：当用户 Holland 主导类型为 A 时，高适配职业多为概念设计师、艺术研究员、品牌策略师等传统艺术/创意路径；若用户或家长担心就业稳定性，可能直接否定整份报告。
- **改进建议 / Suggestion**：在艺术型结果中，将「就业友好备选」提升到更醒目位置（如报告概览下方），并优先展示 UX 设计、内容运营、教育产品设计师等商业化创意岗位。
- **实施状态 / Status**：已在报告概览下新增「稳妥入口职业 | Safe Entry Points」模块，并在就业友好备选池中为 A 型用户提供 UX 设计师、品牌策略师、内容运营、出版编辑、教育产品设计师等商业化或稳定方向。

### 2. [高] 就业友好备选路径位置不够突出 ✅ 已实施

- **类别 / Category**：信息架构
- **问题描述 / Description**：当前备选路径位于「推荐职业路径」章节的最后，用户在快速浏览时可能只看完高/中适配就停止，错过更稳妥的方向。
- **改进建议 / Suggestion**：在报告概览或学生版摘要中直接列出 1-2 个「稳妥入口职业」，并在每个推荐职业卡片上标注「高冒险/高创意」或「稳就业/高需求」标签。
- **实施状态 / Status**：已在报告概览下新增「稳妥入口职业 | Safe Entry Points」模块；所有推荐职业均增加「稳就业型 / Steady Employment」「探索型 / Explore-oriented」「高挑战型 / High Challenge」标签。

### 3. [中] 就业友好备选职业固定，缺乏个性化 ✅ 已实施

- **类别 / Category**：多样性
- **问题描述 / Description**：当前备选职业仅按 RIASEC 类型固定 3 个，不考虑用户的第二、第三码或 CliftonStrengths 优势领域。
- **改进建议 / Suggestion**：根据 Holland 三码和 CliftonStrengths 领域分布，为每个用户动态生成更个性化的备选路径。
- **实施状态 / Status**：已重构备选池，优先匹配「主导 RIASEC × 主导 Domain」，其次匹配「主导 RIASEC × 次要 Domain」或「次要 RIASEC × 主导 Domain」，实现按用户测评结果动态推荐。

### 4. [中] 推荐专业数量可能过多，使用者难以聚焦 ✅ 已实施

- **类别 / Category**：可读性
- **问题描述 / Description**：高适配+中适配+备选路径累计专业可达 30+，对高中生/大学生来说信息过载。
- **改进建议 / Suggestion**：将推荐专业按优先级分组（核心专业 5-8 个 / 拓展专业 5-8 个 / 可关注专业），并在学生版只展示核心专业。
- **实施状态 / Status**：已在报告中新增「推荐申请专业 | Recommended Majors」章节，将专业分为「核心 / Core」「拓展 / Expand」「可关注 / Watch」三层，每层上限 8/8/6 个。

### 5. [中] 主题叙事描述中英文混杂，影响报告专业感 🔄 部分实施

- **类别 / Category**：内容质量
- **问题描述 / Description**：素材库中部分主题为英文原文，部分为中文翻译，导致同一份报告出现中英夹杂。
- **改进建议 / Suggestion**：统一翻译全部 34 个主题的 description、application、blind_spots 为中文（或英文），保持报告语言一致。
- **实施状态 / Status**：已为全部 34 个主题补充统一的双语一句话定义（Top 5 表格）；完整 narrative 翻译与标准化列为后续版本任务，当前在附录中明确说明素材语言差异。

### 6. [中] 部分职业描述仍偏抽象或简短 ✅ 已实施

- **类别 / Category**：内容质量
- **问题描述 / Description**：虽然已补充具体工作内容，但部分描述仍较笼统，如「企业高管」「政治家」「投资人」等，难以让用户形成画面。
- **改进建议 / Suggestion**：对所有职业描述进行质量审核，确保每条描述都包含具体、可感知的日常工作内容；对管理层/领导类职业补充典型任务与决策场景。
- **实施状态 / Status**：已在脚本中为「企业高管」「政治家」「投资人」「创业者」「销售总监」「项目经理」「运营总监」等抽象职业补充更具体的工作内容描述；其余职业描述已纳入现有素材库。

### 7. [中] 推荐专业未结合目标留学地区实际专业设置 ✅ 已实施

- **类别 / Category**：实用性
- **问题描述 / Description**：报告中的推荐专业名称较为通用，未区分国内升学与海外申请场景。对于计划留学的用户，「课程与教学论」「组织发展」「技术营销」等名称与英、澳、加、港高校的课程名称（Programme/Course Title）和学位名称（Degree Title）不直接对应，容易导致选校和文书定位偏差。
- **改进建议 / Suggestion**：
  - **国内路径**：建立专业名称与中国教育部《普通高等学校本科专业目录》的映射，提供可填报的专业代码与院校示例。
  - **海外路径（英/澳/加/港）**：为每个推荐职业补充对应留学地区的常见专业名称。
  - 在报告生成时根据用户选择的申请方向动态渲染专业推荐列表。
- **实施状态 / Status**：已在报告中新增「推荐申请专业 | Recommended Majors」章节，每个专业同时展示「国内可报专业说明」与「海外常见方向（UK/AU/CA/HK）」；并引入国内专业别名映射和海外专业映射。

### 8. [中] 报告未提供就业市场数据辅助决策 🔄 部分实施

- **类别 / Category**：实用性
- **问题描述 / Description**：使用者无法从报告中判断推荐职业的就业前景、薪资水平、入行门槛、地域分布。
- **改进建议 / Suggestion**：引入第三方就业数据（如麦可思、智联招聘、BOSS直聘），为每个推荐职业补充就业率、薪资、地区分布、学历要求等。
- **实施状态 / Status**：已在报告中新增「就业市场参考 | Employment Market Reference」占位模块，列出可接入的数据项与参考来源；实际数据接入需后续对接第三方数据源。

### 9. [低] 报告篇幅较长，学生版不够一页纸 ✅ 已实施

- **类别 / Category**：可读性
- **问题描述 / Description**：当前报告约 240 行，学生版与专业版合并输出，学生阅读负担较重。
- **改进建议 / Suggestion**：拆分为「学生一页纸摘要」和「专业详细报告」两个独立文件；学生版只保留核心结论、3-5 个方向与 3 个行动。
- **实施状态 / Status**：已拆分为 `sample-report-student.md`（一页纸摘要）和 `sample-report-professional.md`（专业详细版），后者包含完整剖面、矩阵、专业映射与行动跟踪模块。

### 10. [低] 模拟得分未考虑 RIASEC 类型之间的相关性 ✅ 已实施

- **类别 / Category**：数据真实性
- **问题描述 / Description**：完全随机生成 6 个得分，可能出现现实中较少见的剖面（如 S=95 同时 E=20），导致报告案例不具代表性。
- **改进建议 / Suggestion**：模拟时可基于真实人群的典型剖面分布生成，或从现有学生数据中抽样。
- **实施状态 / Status**：已在脚本中引入 8 组典型学生 RIASEC 剖面模板，模拟时基于模板添加适度随机扰动，生成更贴近现实的得分分布。

### 11. [低] 交叉矩阵仅标红主导象限，未体现第二、第三码的影响 ✅ 已实施

- **类别 / Category**：可视化
- **问题描述 / Description**：Holland 三码的完整信息未在矩阵中可视化展示，用户只看到第一码的交叉点。
- **改进建议 / Suggestion**：在矩阵中同时标注主导象限、次要象限，或用热力图显示三码加权后的适配度。
- **实施状态 / Status**：已在交叉矩阵中增加三级标注：粗体 = 主导交叉象限；⭐ = 次要交叉象限；○ = 第三码相关象限。

### 12. [低] 所有映射证据等级均为 C/D，缺乏 A/B 级证据 ✅ 已实施

- **类别 / Category**：科学性
- **问题描述 / Description**：报告中大量出现 C/D 证据等级，可能降低用户信任感。
- **改进建议 / Suggestion**：明确说明这是理论推导工具；未来可通过追踪用户就业去向积累 A/B 级证据。
- **实施状态 / Status**：已在专业报告中新增「证据等级说明与科学性建设」章节，定义 A/B/C/D 四级证据标准，并列明毕业生追踪、专家评议、用人单位调研、纵向对照实验四条 A/B 证据建设路径。

### 13. [低] 下一步行动缺乏后续跟踪机制 ✅ 已实施

- **类别 / Category**：落地性
- **问题描述 / Description**：报告列出行动建议，但没有让用户记录完成情况或复盘感受的模块。
- **改进建议 / Suggestion**：在在线工具中增加「探索行动打卡」「职业访谈记录」「专业体验日志」等功能，形成闭环。
- **实施状态 / Status**：已在专业报告中新增「行动跟踪模块 | Action Tracking Module」，包含职业访谈记录、专业体验记录、月度复盘三张表格；在线版可进一步实现自动保存与提醒。
